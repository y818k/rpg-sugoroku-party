# RPG Sugoroku Party

ブラウザで遊べる、最大4人対戦のRPG風すごろくゲームMVPです。Next.js + TypeScript + Socket.IO で、1台PC内の複数ブラウザ/タブ、または同一Wi-Fi/LAN内のスマホからルーム参加して遊べます。

## 起動方法

```bash
npm.cmd install
npm.cmd run dev
```

起動後、PCでは以下を開きます。

```text
http://localhost:3000
```

別タブや別ブラウザで同じURLを開き、ルームコードを入力すると最大4人まで参加できます。

## スマホ実機テスト手順

このアプリはデフォルトで `HOST=0.0.0.0` として起動します。Socket.IO も `io()` の相対接続を使っているため、`localhost` 固定ではありません。

1. PCとスマホを同じWi-Fi、または同じLANに接続します。
2. PCでサーバーを起動します。

```bash
npm.cmd run dev
```

明示的に指定したい場合は、Windows PowerShellで以下を実行します。

```powershell
$env:HOST="0.0.0.0"; $env:PORT="3000"; npm.cmd run dev
```

または cmd.exe では以下です。

```bat
set HOST=0.0.0.0&& set PORT=3000&& npm.cmd run dev
```

3. PCのIPアドレスを確認します。

```powershell
ipconfig
```

`IPv4 Address` を探してください。例:

```text
192.168.0.5
```

4. スマホのブラウザで以下を開きます。

```text
http://PCのIPアドレス:3000
```

このPCでは、現在のLAN IPが `192.168.0.5` の場合は以下です。

```text
http://192.168.0.5:3000
```

5. PC側ブラウザでルームを作成し、スマホ側で同じルームコードを入力して参加します。

### 接続できない場合

- PCとスマホが同じWi-Fi/LANにいるか確認してください。
- VPN、ゲストWi-Fi、モバイル回線では相互接続できないことがあります。
- Windows Defender Firewall が Node.js の通信をブロックしている場合があります。許可ダイアログが出たら、プライベートネットワークで許可してください。
- 3000番ポートが別アプリで使用中の場合は、`PORT` を変えて起動してください。

例:

```powershell
$env:HOST="0.0.0.0"; $env:PORT="3001"; npm.cmd run dev
```

スマホからは以下でアクセスします。

```text
http://PCのIPアドレス:3001
```

## MVP上の割り切り

- ルーム状態はサーバープロセスのメモリ上に保持します。サーバー再起動で消えます。
- 外部DB、ログイン、課金、広告、チャットは未実装です。
- 切断プレイヤーのAI操作は未実装で、MVPでは切断中表示と再接続に対応しています。
